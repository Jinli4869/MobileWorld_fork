# Phase 3.1: Metrics Instrumentation & KPI Contracts - Research

**Researched:** 2026-04-17
**Domain:** KPI schema, per-step telemetry, TTFT fallback policy, latency/reliability aggregation
**Confidence:** HIGH

## Summary

Current runtime already stores cumulative token usage but lacks first-class KPI schema, step-level token delta, latency distribution aggregation, and reliability metrics. Phase 3.1 can be delivered incrementally by:

1. Adding canonical metrics event schema and logger hooks.
2. Introducing a metrics collector that derives per-step token deltas from cumulative usage and computes estimated TTFT/TTFA/TTS.
3. Persisting run-level KPI summaries to artifacts and eval report payloads.
4. Adding contract tests for quality flags and metrics artifact consistency.

This preserves backwards compatibility while making metrics comparable across built-in and adapter runs.
